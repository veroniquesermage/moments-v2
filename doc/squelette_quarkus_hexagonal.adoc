== Communication entre modules

=== Program consomme Person

....
program.application.provider.ParticipantProvider      # interface
program.infrastructure.adapter.PersonParticipantAdapter # classe
person.application.reader.PersonReader                # interface
....

Flux :

....
CreateActivityUseCase
    -> ParticipantProvider
        -> PersonParticipantAdapter
            -> PersonReader
....

Le module `program` ne manipule pas `person.domain.Person`.

Il manipule uniquement :

....
program.domain.Participant
....

Le module `person` expose uniquement :

....
person.application.reader.PersonReader
person.application.reader.PersonDisplay
....

== Règles de dépendance

....
domain
  ne dépend de rien

application
  dépend de domain
  contient les interfaces nécessaires au module

infrastructure
  dépend de application et domain
  implémente les repositories et les adapters

presentation
  dépend de application
  transforme DTO API vers command/use case
....

== Règle FK

....
FK autorisée dans un même module.
FK interdite entre deux modules.
....

Exemple autorisé :

....
program_activity_assignment.activity_id
    -> program_activity.id
....

Exemple interdit :

....
program_activity_assignment.participant_id
    -> person_person.id
....

À la place :

....
program_activity_assignment.participant_id UUID
....

== Règle suppression

....
Une personne référencée n’est pas supprimée physiquement.
Elle passe à INACTIVE.
....

Pour assigner une personne à une activité :

....
Program -> ParticipantProvider -> PersonReader.getActivePersonForAssignment(...)
....

Pour afficher une ancienne activité :

....
Program peut demander une vue d’affichage même si la personne est inactive.
....
